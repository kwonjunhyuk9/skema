# 의사결정 기록

## DL-001: 프로젝트 초기 구조 확립 (2025.03 ~ 2025.04)

**문제**

- 빠른 프로토타이핑과 정보 구조 설계가 우선적으로 필요함

**결정**

- HTML/CSS 기반의 정적 웹 구조로 초기 리포지토리 구성

**근거**

- 초기 단계에서 기술 부채를 최소화할 필요가 있음
- 정적 구조에서 UI/UX와 정보 구조 검증이 용이함

**결과**

- 기본 HTML/CSS 레이아웃 확보
- 향후 React/Next로 확장 가능한 기반 마련

---

## DL-002: 콘텐츠 중심의 블로그 시스템 확장 (2025.04 ~ 2025.05)

**문제**

- 블로그 데이터가 부족하여 전체 서비스 구조를 검증하기 어려움

**결정**

- 블로그 콘텐츠 데이터를 대량 추가하여 구조를 선제적으로 확장

**근거**

- 블로그는 트래픽을 유발하는 핵심 콘텐츠 영역임
- 정적 데이터 기반 개발은 구현 및 배포 효율성이 높음

**결과**

- 블로그 카테고리 구조 체계화
- 콘텐츠 중심 확장을 고려한 데이터 모델 확립

---

## DL-003: React/Next 프론트엔드 도입 (2025.07 ~ 2025.07)

**문제**

- 정적 웹사이트 구조로는 사용자별 데이터 개인화가 불가능함

**결정**

- HTML/CSS 기반 구조를 React/Next 기반의 동적 웹사이트로 마이그레이션

**근거**

- 컴포넌트 재사용성 필요
- 라우팅 및 상태 관리 구조 개선 필요
- SEO 최적화 요구

**결과**

- 프로젝트 전체 구조 재설계
- 확장 가능한 디렉터리 및 파일 구조로 전면 개편

---

## DL-004: Vercel 호스팅 도입 (2025.07 ~ 2025.07)

**문제**

- 프로젝트를 외부에 배포할 호스팅 환경이 존재하지 않음

**결정**

- Vercel을 공식 호스팅 및 배포 플랫폼으로 채택

**근거**

- Next와의 높은 호환성
- CI/CD 자동화 및 간편한 배포 환경 제공
- 무료 플랜에서도 충분한 초기 운영 가능

**결과**

- 프로덕션 환경 배포 완료
- 배포 및 롤백 프로세스 단순화

---

## DL-005: UI/UX 전면 개편 (2025.07 ~ 2025.07)

**문제**

- React/Next 마이그레이션 이후 UI 일관성이 부족함

**결정**

- 디자인 시스템을 재정의하고 전체 UI/UX 구조를 전면 개편

**근거**

- 사용자 경험 품질 향상 필요
- 모바일 접근성 및 반응형 대응 강화 요구

**결과**

- 재사용 가능한 UI 컴포넌트 구조 수립
- 네비게이션, 스크롤바, 사이드바 UI 통합 및 개선

---

## DL-006: 문서화 강화 및 프로세스 명확화 (2025.07 ~ 2025.08)

**문제**

- 프로젝트 규모 확장에 따라 구조와 의사결정의 명확성이 부족함

**결정**

- 요구사항 정의서, 시스템 아키텍처 문서를 작성하여 문서 중심 개발 프로세스 도입

**근거**

- 기술적 의사결정의 일관성 유지 필요
- 유지보수 및 협업 가능성 확보 필요

**결과**

- Documentation-Driven Development 기반 마련
- 프로젝트 이해도 및 확장성 향상

- 실시간 API 및 인증 기능 통합
